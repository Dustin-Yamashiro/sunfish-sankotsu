## 5. ブランド方針

- トーン: 穏やか、透明感、清潔感、やさしさ、信頼感、石垣島らしい自然感
- キーワード: 海、空、花、白い余白、水色、静けさ、見送り、相談しやすさ
- ロゴ: ヘッダー左、フッターで使用。水色の波・円形モチーフを想定。
- 写真: 海、水面、花、船、人の後ろ姿を主役にする。均一グリッドではなく、大小差・ずらし・重なりで構成する。
- イラスト: 「散骨の流れ」のステップ円内など、控えめな線画アイコンを想定。
- アイコン: 問い合わせ、LINE、電話、ステップ、矢印など。装飾過多にしない。

## 6. デザインの方向性

- 白背景と広い余白を基調に、海・空・花の写真で透明感と穏やかさを作る。
- 水色、白、薄い水色を中心に、清潔感と信頼感を出す。
- 写真は均等なカードグリッドではなく、大小差・ずらし・重なりで自然なリズムを作る。
- CTAは問い合わせとLINE相談を明確に分ける。
- 装飾よりも写真、余白、読みやすい本文を優先する。

- 写真背景上: `#FFFFFF` の文字と白い円形要素を基本にし、可読性を確保する。

## 8. グラデーション

水色グラデーションは右上から左下へ流す。

```css
linear-gradient(
  to bottom left,
  #5FBFD3 0%,
  #78BFCD 50%,
  #84C7D3 75%,
  #92CFD8 88%,
  #9AD5DB 100%
)
```

- 名称: 水色グラデーション
- 用途: 水色面、背景表現候補
- 注意: 白背景の余白セクションへ多用しない。

## 9. タイポグラフィ

| 役割 | フォント | 目安 | 用途 |

| 英字補助タイトル | Cormorant Garamond / Bold | 15px中心、`#76BDD0` または白 | About、Plan、Step、FAQ |
| 本文 | Noto Sans JP / Regular-Medium | 16px基準、行間1.7前後 | 説明文、FAQ本文 |

- 通常文字色は `#333333`。
- 写真背景上は白文字を基本にする。
- 本文は長くなりやすいため、見た目の繊細さより可読性を優先する。
- 字間は見出し・英字補助でやや広め、本文は読みやすさを優先する。
- 通常の可読テキストは16px以上を基準にし、15px以下は英字補助、日付、注釈、装飾的な小要素など、デザイン上必要な場合だけ使う。

## 11. セクション別ルール

### ファーストビュー

- 海の写真を主役にする。
- ヘッダーは約100pxの白ベース。
- CTAは問い合わせとLINE相談を右側に明確に置く。
- H1やコピーは写真の可読性を損なわない位置に置く。

### 白背景セクション

- 余白を広く取り、文章と写真の呼吸感を出す。
- 写真は均一なカードにせず、大小差やずらしを使う。
- 見出しは英字補助 + 和文タイトルを基本にする。
- メッセージ、海洋散骨とは、FAQなどで使う。

### 薄い水色背景セクション

- `#F0F5F6` を使用する。
- セクション間の静かな区切りや、ニュース外枠に使う。
- 強い装飾は避ける。

### 水色面 + 写真/テキスト重ね

- `#76BDD0` 系の面を大きく使い、白い情報面や写真を重ねる。
- カードUIではなく、横長の情報ブロックとして扱う。
- 文字量が増えても余白が詰まりすぎないようにする。
- 特徴、散骨プラン紹介などで使う。

### 写真コラージュ

- 大小差、上下左右のずらし、重なりを使う。
- 均等なカードグリッドにしない。
- 影や装飾を強くせず、余白と配置で見せる。
- `object-fit: cover` を使う場合は、人物や花が不自然に切れないか確認する。

### 散骨の流れ

- 海と花の写真を全面背景にし、強めにぼかす。
- 前景は白文字、白円、矢印、ステップ番号で構成する。
- 中央の補足コピーは白い短冊背景 + 水色文字。
- ステップ円はカードではなくステップUIとして扱う。
- blur radius の正確値は未確認のため、スクリーンショットに合わせて調整する。

### FAQ

- 中幅コンテナを使い、白背景に静かに配置する。
- 線色・線幅の正確値は未確認。
- 開閉UIはアクセシビリティを優先し、キーボード操作と `aria-expanded` を考慮する。

### お知らせ / コラム

- カードの見た目、画像比率、余白、カルーセル/インジケータはSWELL標準を優先する。
- 子テーマ側では背景、見出し、セクション外枠程度に留める。
- 独自カード化しない。

## 13. 写真表現

- 海、水面、花、船、人の後ろ姿を中心にする。
- 写真は角丸なし、または最小限を基本にする。
- 影や装飾を強くせず、余白と配置で見せる。
- 写真コラージュは均等なグリッドにしない。
- 写真コラージュの写真枠は、原則 `width` と `aspect-ratio` で形を定義し、任意の `width` + `height` 固定だけで縦横を決めない。
- 写真差し替えが想定される配置用の modifier は、写真内容に依存する `--beach` などではなく、`--01`, `--02` のようなスロット名を使う。
- 横長写真帯や背景写真は画面幅を超えて広がりを作る場合がある。
- 投稿/ニュース画像はSWELL側のカードデザインに従うため、本ファイルでは比率・トリミング・枠を定義しない。

## 14. SWELLとの役割分担

- SWELLに任せる: 投稿一覧、ニュース/コラムカード、記事詳細の基本表示、管理画面側の投稿運用。
- 子テーマで作る: トップページ固有セクション、サービスページ固有レイアウト、写真コラージュ、水色面の重ね、CTA配置、散骨の流れ。
- 実装前に、SWELL標準で足りる箇所と子テーマで上書きする箇所を分ける。

## 15. 動き

- ホバー: 単純な色変化、線の変化、矢印の小移動はCSSで実装する。
- 画面表示時フェードイン: 控えめに使う。動きが主役にならないようにする。
- スクロール演出: 写真コラージュや水色面の重なりで必要な場合のみ検討する。
- パララックス: 海・花の写真背景で要件が出た場合のみGSAPで検討する。
- スライダー: 投稿/コラムや写真スライダーが必要な場合はSplideを標準候補にする。
- 自動再生: 必要な場合だけ採用し、停止や `prefers-reduced-motion` を考慮する。
- GSAP / Splide の使用予定: 複数要素の連動、ScrollTrigger、スライダー要件がある場合のみ使用する。

## 16. 素材メモ

- 元画像配置: `assets/images`
- build後画像: `themes/swell_child/assets/images`
- jpg/png/jpeg は build 時に WebP 変換する。
- PHPからの画像参照は `theme_image_url()` を使う。
- 同名異拡張子の画像は WebP 出力名が衝突するため避ける。
- 添付トップページは全体のトーン確認用。セクション実装時は追加スクリーンショットを優先する。
- ロゴやアイコンの元データ、使用権、SVG化の可否は実装前に確認する。
