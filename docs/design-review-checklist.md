# デザイン確認チェックリスト

Figma やスクリーンショットから実装前に読み取る項目です。

## 全体

- [ ] ロゴ、サイト名、メイン CTA がファーストビューで明確か。
- [ ] ヘッダーの固定有無、スマホメニュー、CTA の配置が分かるか。
- [ ] カラーの役割がベース、メイン、アクセント、テキスト、線に整理できるか。
- [ ] フォント、字間、行間、太さの規則があるか。
- [ ] セクション余白と最大幅に一貫性があるか。

## レスポンシブ

- [ ] PC とスマホの並び替えルールが分かるか。
- [ ] 画像比率を固定すべき箇所があるか。
- [ ] ボタンやナビ文言が折り返しても破綻しないか。
- [ ] 固定ヘッダーがコンテンツを隠さないか。

## コンポーネント

- [ ] ボタンの種類、hover、disabled 相当の状態が整理できるか。
- [ ] 見出し、カード、FAQ、ニュース一覧、フォーム部品の共通化余地があるか。
- [ ] 写真の重なり、角丸、影、線、背景色が component か project か判断できるか。

## WordPress

- [ ] 投稿で管理すべき内容と固定テンプレートで持つ内容が分かれるか。
- [ ] カスタム投稿、カテゴリ、タグ、カスタムフィールドが必要か。
- [ ] エディタで編集させる範囲が明確か。
- [ ] 画像 alt、リンク先、CTA 文言を管理する場所が決まっているか。

## 素材

- [ ] jpg/png は WebP 変換後の衝突名がないか。
- [ ] SVG はインライン化するか、画像として扱うか。
- [ ] ロゴやアイコンの権利と元データがあるか。
- [ ] 画像のトリミング位置と `object-fit` が分かるか。

## 品質確認

- [ ] build が通るか。
- [ ] PHP 構文チェックが通るか。
- [ ] 主要ページでブラウザ console error がないか。
- [ ] Lighthouse で明確な画像サイズ、alt、CLS 問題がないか。
- [ ] deploy dry-run の差分が意図通りか。
